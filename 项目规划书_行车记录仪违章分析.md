# 安卓端侧行车记录仪违章分析系统 - 项目规划书

## 1. 项目概述

### 1.1 项目名称
**DashCam Violation Analyzer (DVA)**

### 1.2 项目定位
一款运行在安卓设备上的端侧视频分析应用，无需网络连接即可对行车记录仪视频进行实时/离线分析，自动识别车辆变道不打灯违章行为，截取关键证据图片并识别车牌信息。

### 1.3 核心价值
- **隐私安全**：所有处理在本地完成，视频不上传
- **便捷高效**：即插即用，支持多种视频格式
- **证据级输出**：无损截图 + 结构化数据，满足举证需求

## 2. 功能规格

### 2.1 核心功能 (MVP)

#### 2.1.1 视频文件管理
| 功能 | 描述 |
|------|------|
| 文件浏览 | 扫描本地文件夹，选择视频进行分析 |
| 格式支持 | MP4、MOV、AVI、MKV（通过MediaCodec解码） |
| 批量处理 | 支持整文件夹批量分析，实时显示进度 |
| 分析记录 | 保存分析历史，避免重复处理 |

#### 2.1.2 变道违章检测
**检测规则定义：**
```
变道不打灯违章 = 车辆跨越车道线 + 转向灯状态为OFF

检测逻辑：
1. 车道线检测 → 确定车辆所在车道
2. 持续追踪车辆轨迹
3. 当检测到车辆横向位移超过阈值(车道宽度的30%)时：
   a. 查询对应时刻转向灯状态
   b. 若转向灯关闭 → 标记为违章
```

**预留扩展接口（策略模式）：**
```kotlin
interface ViolationDetector {
    val violationType: ViolationType
    fun detect(frame: Frame, context: DetectionContext): ViolationResult?
}
```

#### 2.1.3 证据截图采集
| 截图类型 | 时间戳 | 要求 |
|----------|--------|------|
| 违章时刻(T0) | 变道发生帧 | 跨越车道线的瞬间 |
| 前序帧(T-5s) | T0 - 5秒 | 变道前的车道位置 |
| 后继帧(T+5s) | T0 + 5秒 | 变道后的车道位置 |

**截图规格：**
- 格式：PNG（无损压缩）
- 分辨率：保持原视频分辨率
- 命名规则：`{视频名}_{车牌号}_{违章类型}_{时间戳}.png`

#### 2.1.4 车牌识别 (LPR)
| 项目 | 规格 |
|------|------|
| 识别范围 | 中国大陆蓝牌/绿牌（新能源） |
| 识别精度 | 针对高清截图优化，目标>95% |
| 输出格式 | 省份简称 + 字母 + 数字 |
| 置信度 | 返回识别置信度，过低时人工复核 |

### 3.2 技术选型

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **UI框架** | Jetpack Compose | 声明式UI，现代化Android开发 |
| **架构** | MVVM + Clean Architecture | 清晰分层，易于测试 |
| **DI** | Hilt | Google官方依赖注入 |
| **视频解码** | MediaCodec + MediaExtractor | 硬件加速，高效解码 |
| **ML推理** | ONNX Runtime Mobile | 高性能跨平台推理 |
| **数据库** | Room | 本地持久化分析记录 |
| **异步** | Kotlin Coroutines + Flow | 响应式处理 |
| **车牌识别** | PaddleOCR-Lite / LPRNet | 轻量级端侧LPR |

### 3.3 模型文件清单

| 模型 | 用途 | 典型大小 | 来源 |
|------|------|----------|------|
| `yolov8n-vehicle.onnx` | 车辆检测 | ~6MB | YOLOv8n 微调 |
| `lanenet.onnx` | 车道线分割 | ~15MB | LaneNet 转换 |
| `lprnet_chinese.onnx` | 车牌识别 | ~10MB | LPRNet Paddle |

## 7. 性能优化策略

| 优化点 | 策略 |
|--------|------|
| **帧采样** | 每秒采样15帧，违章帧前后提升至30fps |
| **并行检测** | 车辆检测 + 车道线检测并行执行 |
| **内存管理** | 双缓冲+对象池，避免GC频繁 |
| **模型量化** | INT8量化模型，推理速度提升2-3倍 |
| **增量分析** | 记录已分析位置，支持断点续传 |
